# 增长漏斗 + 用户分层 + 数据迭代

用来诊断"哪个环节出了问题"和"不同用户怎么区别对待"。

---

## AARRR 增长漏斗

每个阶段对应一个核心问题和关键指标。

### 1. Acquisition 获取
**核心问题**：用户从哪来？
**关键指标**：渠道流量、获客成本（CAC）、渠道转化率
**常见策略**：
- 内容营销（小红书/公众号/短视频）
- 裂变活动（见 viral-growth.md）
- 投放（信息流、SEM）
- 合作互推

**诊断**：流量少 → 渠道不对或内容没吸引力；流量大但质量差 → 钩子和业务不匹配

### 2. Activation 激活
**核心问题**：用户来了之后做了关键动作吗？
**关键指标**：注册完成率、首次核心行为完成率（如完成测评）、跳出率
**常见策略**：
- 简化注册/首次体验流程
- 新手引导设计
- 首次使用奖励
- 减少信息收集（延迟到信任建立后）

**诊断**：注册后没用 → 上手门槛太高或价值感知不够快

### 3. Retention 留存
**核心问题**：用户会回来吗？
**关键指标**：次日留存、7日留存、30日留存、DAU/MAU
**常见策略**：
- 推送/提醒机制
- 持续的内容更新
- 社群运营维系
- 习惯养成设计（打卡、积分）

**诊断**：激活率高但留存差 → 产品没有持续价值，或者首次体验是峰值之后全是下坡

### 4. Revenue 收益
**核心问题**：用户愿意付费吗？
**关键指标**：付费转化率、ARPU（每用户收入）、LTV（生命周期价值）
**常见策略**：
- 免费→付费的价值阶梯设计
- 限时优惠/首单折扣
- 增值服务分层
- 社会证明（付费用户案例）

**诊断**：用得多但不付费 → 免费部分给太多了，或付费价值感知不清晰

### 5. Referral 推荐
**核心问题**：用户会推荐给别人吗？
**关键指标**：NPS（净推荐值）、裂变系数K值、邀请转化率
**常见策略**：
- 产品内分享机制（测评结果海报）
- 转介绍奖励
- 口碑驱动（把产品做到超预期）
- 情感裂变设计

**诊断**：用户满意但不分享 → 缺少分享触发点，或分享路径太复杂

## RARRA 变体（存量市场优先）

当市场增量见顶，获客成本飙升时，优先级调整为：

```
Retention（留存）→ Activation（激活）→ Referral（推荐）→ Revenue（收益）→ Acquisition（获取）
```

**核心思路**：先留住现有用户，再让他们带来新用户，最后才考虑花钱买量。

## RFM 用户分层

三个维度：
- **R** (Recency)：最近一次互动/购买距今多久
- **F** (Frequency)：互动/购买频率
- **M** (Monetary)：消费金额/互动深度

### 八种用户类型 + 运营策略

| R | F | M | 用户类型 | 运营策略 |
|---|---|---|---------|---------|
| 高 | 高 | 高 | 重要价值用户 | VIP服务，专属权益，优先维护 |
| 高 | 低 | 高 | 重要发展用户 | 提频次：推荐更多场景和产品 |
| 低 | 高 | 高 | 重要保持用户 | 召回：专属优惠、关怀问候 |
| 低 | 低 | 高 | 重要挽留用户 | 紧急召回：大力度优惠+1对1沟通 |
| 高 | 高 | 低 | 一般价值用户 | 提客单价：推荐升级版/增值服务 |
| 高 | 低 | 低 | 一般发展用户 | 培养习惯：低门槛活动拉频次 |
| 低 | 高 | 低 | 一般保持用户 | 轻度召回：推送内容维系关系 |
| 低 | 低 | 低 | 流失用户 | 低成本触达，不值得重投入 |

### 非电商场景的 RFM 变体

不是所有业务都有"消费金额"。可以替换M维度：
- 内容产品：M = 阅读深度/互动次数
- 测评产品：M = 完成测评数/分享次数
- 社群产品：M = 发言频率/帮助他人次数

## 数据迭代闭环

### 看什么数据

```
渠道 → 落地页 → 注册/参与 → 核心行为 → 付费 → 推荐
  ↓        ↓         ↓           ↓         ↓       ↓
曝光量   跳出率   完成率     活跃率    转化率   K值
```

### 判断哪步出问题

| 现象 | 可能原因 | 排查方向 |
|------|---------|---------|
| 曝光高点击低 | 标题/封面不吸引 | A/B测试不同标题 |
| 点击高跳出高 | 落地页和预期不符 | 检查标题承诺 vs 页面内容 |
| 注册率低 | 表单太长/要太多信息 | 砍字段，延迟收集 |
| 激活率低 | 上手太复杂/价值感知慢 | 简化首次体验，前置结果预览 |
| 留存差 | 没有持续价值/缺提醒 | 增加内容更新/推送 |
| 付费率低 | 免费给太多/付费感知弱 | 调整免费/付费边界 |
| 分享率低 | 缺触发点/路径复杂 | 加一键分享+分享激励 |

### AB测试基本原则
1. 一次只改一个变量
2. 样本量足够（至少各组100+）
3. 跑够时间（至少24小时覆盖不同时段）
4. 看统计显著性，不看"感觉"

## 什么时候不要用漏斗思维

- 产品还没找到PMF时，别急着优化漏斗——先确认用户到底要什么
- 数据量太小时（日活<100），数据波动太大，看数据不如做用户访谈
- 过度优化某个环节可能损害整体体验（如为了提注册率去掉所有表单字段，导致后续无法做个性化）
