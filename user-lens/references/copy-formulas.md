# 文案公式 + 钩子设计

写文案时的结构模板。**公式是骨架，不是填空——用完之后要打磨成自然的人话。**

---

## 三大文案公式

### PAS — 痛点型（适合海报、朋友圈、短文案）

```
P (Problem)  → 指出用户正在经历的痛
A (Agitate)  → 搅动痛感，让用户觉得"不解决不行"
S (Solution) → 给出解决方案
```

**示例**（MBTI留学测评）：
- P：问了10个人去哪留学，拿到10个不同答案
- A：因为他们都不了解你的性格。选错方向比不努力更可怕——转专业要浪费一整年
- S：5分钟测出你的留学性格，直接匹配最适合你的国家和专业

**适用场景**：海报文案、落地页首屏、社群话术、朋友圈广告

---

### AIDA — 漏斗型（适合落地页、长文案、公众号推文）

```
A (Attention) → 抓注意力（标题/首屏）
I (Interest)  → 引发兴趣（展开痛点或好奇心）
D (Desire)    → 激发欲望（展示收益、案例、社会证明）
A (Action)    → 引导行动（明确CTA）
```

**示例**：
- Attention：选错专业=白读一年，你的性格真的适合国外课堂吗？
- Interest：很多同学的问题不是努力不够，而是方向选错了
- Desire：32道题测出15个维度，已有1000+留学生验证准确率
- Action：扫码5分钟免费测评，看看性格替你选了哪条路

**适用场景**：公众号推文、落地页、详情页、邮件营销

---

### FAB — 产品型（适合产品介绍、详情页）

```
F (Feature)   → 产品特点（有什么）
A (Advantage) → 相比竞品的优势（好在哪）
B (Benefit)   → 对用户的好处（跟我有什么关系）
```

**示例**：
- F：32道留学场景题，覆盖15个性格维度
- A：不是通用MBTI，是针对留学决策定制的
- B：测完直接知道你适合去哪个国家、学什么专业，不用再纠结

**适用场景**：产品详情页、功能介绍、销售话术

**注意**：FAB是产品介绍文案，不是转化文案。用FAB写完后，要用PAS/AIDA的逻辑包装成用户视角。

## 产品介绍 → 产品包装 转化方法

这是从培训文档提炼的核心方法。产品介绍说的是"我有什么"，产品包装说的是"你怕什么/你要什么"。

### 转化公式

```
产品功能 → 用户场景 + 用户情绪 + 用户语言
```

### 对照表

| 产品介绍（不好） | 产品包装（好） | 转化逻辑 |
|----------------|--------------|---------|
| 提供性格测评及国家、专业推荐 | 「选错专业=白读一年」测出你最不该选的方向 | 功能→恐惧场景 |
| 提供学习能力与社交能力分析 | 「你能不能适应国外课堂？」3分钟测出真实适配度 | 功能→用户自问 |
| 32道题覆盖15个维度 | 比问10个人都准，5分钟看清自己 | 参数→对比感知 |
| 匹配最适合的国家和专业 | 你的性格早就替你选好了方向，只是你还不知道 | 功能→悬念钩子 |

### 转化心法
1. **不要从产品出发写文案**——先想用户在什么场景下看到，当时什么心情
2. **用用户会说的话**——"纠结"而不是"决策困难"，"坑"而不是"风险"
3. **一个卖点说透 > 五个卖点罗列**——海报只需一个核心打击点
4. **具体 > 抽象**——"32道题5分钟"比"快速精准测评"有感觉

## 标题公式

### 公式一：数字 + 痛点 + 解决方案
- "3分钟测出你最不该选的留学方向"
- "留学前必做的1件事，比看100个排名都有用"

### 公式二：反常识 + 好奇心
- "选校最重要的不是排名，是弄清楚你是哪种人"
- "你的MBTI早就替你选好了留学方向"

### 公式三：身份代入 + 场景
- "闺蜜选了英国，室友选了美国，你在纠结港新澳"
- "问了中介、问了学长、刷了100条帖子，还是不确定"

### 公式四：恐惧/损失
- "选错国家比选错专业还难调头"
- "选错方向浪费的不是钱，是你最好的两年"

### 公式五：社会证明
- "1000+留学生测完说：早知道就不纠结了"
- "已有XX人解锁了自己的留学性格密码"

## 钩子设计框架

钩子 = 让用户愿意参与/分享/付费的那个"诱饵"。

### 四个必要条件

| 条件 | 怎么判断 | 不满足会怎样 |
|------|---------|------------|
| 稀缺性 | 网上搜不到/买不到同类 | 用户觉得"这我自己也能找到" |
| 社交形象 | 分享到朋友圈不丢脸 | 用户想要但不敢分享 |
| 业务相关 | 吸引来的人是目标用户 | 来了一堆薅羊毛的 |
| 获得感 | 拿到后马上有收获感 | 用户拿了不用，后续转化为零 |

### 钩子价值公式

```
用户感知价值 > 用户付出成本 → 参与
用户感知价值 < 用户付出成本 → 放弃
```

付出成本包括：时间、社交成本（转发到朋友圈的心理负担）、隐私成本（填手机号）

### 降低成本的技巧
- 只收微信号不收手机号（用户觉得更安全）
- 先给部分结果，再引导解锁完整版（禀赋效应）
- 多级阶梯：拉2人得A，拉5人得B（降低放弃率）
- 把"转发"包装成"分享给需要的朋友"（降低社交成本）

---

## 什么时候不要套公式

- 用户已经很熟悉你（老粉丝、已付费用户）→ 直接说人话，不需要钩子
- 产品本身就有强需求（刚需产品）→ 说清楚是什么就够了
- 文案已经写了三版还在套公式 → 放下公式，回去做用户访谈，问题可能不在文案
