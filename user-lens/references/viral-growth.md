# 裂变增长机制

从设计到执行的裂变全流程。**裂变不是万能的——没有存量用户、没有好钩子，再巧的机制也跑不起来。**

---

## 裂变的本质

传统广告：先付费 → 再传播 → 受益者是渠道
裂变：老用户推荐新用户 → 成功后奖励 → 受益者是用户 → 费用后置

核心公式：**用户行动 = 动力 × 能力 × 提醒**
- 动力：钩子够不够吸引
- 能力：流程够不够简单
- 提醒：触达够不够及时

---

## 六种裂变模式

### 1. 分享裂变（入门级）
用户分享链接/海报到朋友圈即完成任务。
- 门槛最低，适合冷启动
- 适用：引流、品牌曝光
- 案例：分享海报到朋友圈集赞，截图领优惠

### 2. 邀请裂变
用户分享后，被分享者需完成指定操作（注册/关注/填表）。
- 门槛中等，用户质量更高
- 适用：拉新、会员增长
- 案例：瑞幸咖啡老带新，双方各得优惠券

### 3. 拼团裂变
多人组团完成购买，享受折扣。
- 适用：电商、知识付费、培训课程
- 案例：拼多多拼团、培训课程团购价

### 4. 分销裂变
用户推荐他人购买，获得佣金。最多二级，超过三级涉嫌传销。
- 适用：知识付费、课程、小额消费品
- 注意：用户可能不好意思"赚朋友钱"——需要KOL先发海报拉高势能

### 5. 众筹裂变（助力裂变）
邀请指定数量好友完成操作，解锁福利。
- 适用：需要精准传播、好友关系较亲密的场景
- 案例：拼多多砍价、拆红包
- 关键：助力人数必须与福利力度匹配

### 6. 情感裂变
没有利益驱动，靠社交货币（成就感、身份认同、情感表达）驱动分享。
- 适用：测评结果、年度报告、成就海报
- 案例：支付宝年度账单、网易云年度歌单、MBTI测评结果
- **这就是我们MBTI留学测评的核心裂变模式**——用户分享测评结果是因为"说的就是我"

---

## 模式选择决策树

```
你的产品是免费的吗？
├── 是 → 用户有社交分享动力吗？
│   ├── 是（测评/报告/成就）→ 情感裂变
│   └── 否 → 分享裂变 + 小奖品激励
└── 否 → 客单价高吗？
    ├── 高（>100）→ 拼团裂变 / 分销裂变
    └── 低（<100）→ 邀请裂变 / 众筹裂变
```

## 裂变三要素

### 1. 存量用户
- 最低门槛：**200个活跃用户**
- 没有存量，裂变跑不起来（农民伯伯的故事）
- 种子用户质量 > 数量：选影响力高、活跃度高的

### 2. 钩子设计（福利激励）

**钩子4条件**：

| 条件 | 说明 | 反面 |
|------|------|------|
| 稀缺性 | 不烂大街，有独特价值 | 网上随便搜到的资料包 |
| 社交形象 | 分享出去不丢脸，最好能装X | 很LOW的东西就算想要也不敢发朋友圈 |
| 业务相关 | 什么钩子吸引什么用户 | 用母婴产品引流知识付费 = 用户不匹配 |
| 获得感 | 简单易上手，马上感到收获 | 给了一堆看不懂的专业资料 |

**钩子价值 > 用户付出成本**，否则用户不参与。

邀请人数建议：3-8人，最多不超过12人。
多级阶梯可以降低放弃率：拉2人得A，拉5人得B，拉12人得C。

### 3. 数据监控
- 海报扫码率 → 判断海报设计
- 扫码后参与率 → 判断活动规则
- 完成率 → 判断任务难度
- 裂变系数（K值）→ 每个用户平均带来几个新用户

## 裂变SOP（执行流程）

### Phase 1：策划
1. **分析受众**：目标用户是谁？在哪？痛点是什么？
2. **选钩子**：调研用户需求（竞品分析、用户访谈、社区调研）
3. **选模式**：根据决策树选裂变类型
4. **设计流程**：画出完整用户路径图

### Phase 2：预热（流量节奏设计）
1. **第一波：KOL发海报** → 拉高势能，让用户觉得"大咖都认可"
2. **第二波：合作流量转发** → 行业媒体/社群扩散，反复提醒
3. **第三波：启动用户裂变** → 用户已被多次触达，参与意愿最高

> 用户需要 **7次提醒** 才能完成一次成交。节奏设计的目的就是在正式启动前积累足够的提醒次数。

### Phase 3：执行 + 小步快跑
1. 先投放到 20-30 个群（3000-5000人）做测试
2. 观察数据：
   - 扫码率低 → 改海报设计/文案
   - 参与率低 → 改活动规则/流程
   - 完成率低 → 降低任务难度/加阶梯奖励
3. 迭代优化后再全量投放

### Phase 4：转化闭环
- 裂变不是终点，**第一时间转化**
- 新用户黏性低，放久了流失率飙升
- 在活动流程中嵌入转化环节（入群→体验课→正价课）

## 裂变海报设计要点

| 要素 | 要求 |
|------|------|
| 主标题 | 戳痛点，用用户语言，不用产品语言 |
| 副标题 | 补充说明"这是什么""对我有什么用" |
| 信任背书 | KOL推荐/用户数/机构名 |
| 紧迫感 | 限时/限量/限人群（要有合理理由） |
| 行动指令 | 明确告诉用户下一步做什么 |
| 二维码 | 足够大，位置显眼 |

---

## 什么时候不要做裂变

- 没有存量用户（先做内容/投放积累种子）
- 产品没有验证过PMF（裂变放大的是产品价值，也放大产品问题）
- 低频高价产品（考虑把产品打碎，用高频低价的子产品做裂变）
- 用户群体社交敏感度高（高端用户不愿意"帮你拉人头"）
