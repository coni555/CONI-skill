# 色板参考

6 组经过验证的 OKLCH 色板结构。**不是菜单——是安全网。**

## 使用方式

正确流程：从内容气质出发推导色相区 → 用这些预设验证 L/C 关系是否健康 → 如果自配色板通过下方诊断表就用自配的。

预设的价值是：
- 展示健康的 L（明度）差距范围
- 展示健康的 C（饱和度）层次
- 提供变量命名规范（`--paper`, `--ink`, `--accent`, `--mute` + 一个特色变量）
- 当你对自配色板没信心时，可以直接用预设兜底

## 硬规则

- 和上一份产物的色相区（Hue）差距 ≥ 60°
- 主色和背景 L 差距 ≥ 0.4
- 强调色 C 远高于主色（至少多 0.10）
- 一份页面最多 4-5 个色彩变量

## 1. 暖纸朱砂（Hue 30°区 · 红橙）

```css
:root {
  --paper:   oklch(0.96 0.02 88);
  --ink:     oklch(0.20 0.02 30);
  --accent:  oklch(0.55 0.18 30);
  --gold:    oklch(0.72 0.15 80);
  --mute:    oklch(0.55 0.02 35);
}
```
适合：70 年代出版物、复古广告策略 deck、人文评论

## 配色诊断

| 问题 | 原因 |
|---|---|
| 整体"糊" | 主色和背景 L 差距 < 0.4，拉大 |
| 没层次 | accent 的 C 和主色太接近，accent C 至少 0.15 |
| 太花 | 用了 > 4 个主色，砍到 3-4 |
| 中文衬线不舒服 | 背景太冷，换暖色系（Hue 30-90） |
| 黑体不精神 | 背景太暖，换中性/冷色（Hue 200-300） |
