# 交互组件指南

页面至少有 1 个真有用的交互——不是 hover 变色，是按动后改变信息的交互。

---

## "真有用"的判断标准

一个交互是否该做，问三个问题：

1. **它让"反常识"变得可触摸吗？**（计算器把"不用一笔还清"变成可拖动滑块）
2. **它让"对比"变得可切换吗？**（颗粒度切换器一键看三档差异）
3. **它让"抽象概念"变得可演示吗？**（MBTI 介绍 vs 包装的对照器）

三个都不是 → 不做这个交互，换一个。

---

## 交互类型库

### 切换器 / Toggle Comparator

**场景**：内容有明显的 A vs B 对照、before/after、不同粒度的展示

```
[按钮A] [按钮B] [按钮C]
┌─────────────────────────┐
│  当前选中项的内容展示      │
└─────────────────────────┘
```

**示例**：
- 同一个卖点的"新手版 / 进阶版 / 专家版"说法对照
- 方案的"成本视角 / 效果视角 / 风险视角"
- 策略的"短期 / 中期 / 长期"

---

### 计算器 / Slider Calculator

**场景**：内容涉及可量化的决策、存在"代入自己的数字就能得出结论"的公式

```
输入: [滑块 ━━━●━━━━━] 当前值: 250万
输出: 月供 ¥12,500 | 总利息 ¥180,000
```

**示例**：
- 贷款/投资收益计算器
- "你的时间值多少钱"计算器
- 项目规模 → 推荐团队配置

---

### 对照器 / Side-by-Side Diff

**场景**：核心论点是"两件事看起来像但本质不同"

```
┌──── 表面相似 ────┐  ┌──── 本质区别 ────┐
│  方案A的描述      │  │  方案B的描述      │
│  ...             │  │  ...             │
└─────────────────┘  └─────────────────┘
         ↕ 点击切换维度
```

**示例**：
- "介绍 vs 包装"的对照
- 好文案 vs 坏文案
- 正确做法 vs 常见误区

---

### 流程可视化 / Step Navigator

**场景**：内容本身是一个多步流程，每步有不同的注意事项

```
① → ② → ③ → ④ → ⑤
        ↑ 当前高亮
┌─────────────────────────┐
│  当前步骤的详细说明        │
│  + 常见错误提示           │
└─────────────────────────┘
```

---

### 自检打分器 / Scorecard

**场景**：内容结尾适合让读者代入自评

```
□ 我能在30秒内说清产品卖点     (+2分)
□ 我的标题有冲突感             (+2分)
■ 我用了数据做支撑             (+2分) ✓
━━━━━━━━━━━━━━━━
当前得分: 4/10 · 段位: 新手
```

---

### 时间轴 / Timeline Scrubber

**场景**：内容涉及时间演变、阶段划分

```
2020 ──●── 2021 ──●── 2022 ──●── 2023 ──●── 2024
              ↑ 拖动查看该阶段详情
```

---

### 层级展开器 / Progressive Disclosure

**场景**：信息有明显的"概览 → 详情 → 细节"层级

```
▸ 策略一：差异化定位        ← 点击展开
  ├ 核心逻辑：...
  ├ 实操步骤：...
  └ 常见陷阱：...
```

---

## 找不到交互？

去原文里找以下信号：

| 信号 | 对应交互 |
|---|---|
| 核心比喻（"把孩子当分红股"） | 围绕比喻做对照器 |
| 反复出现的公式（"利益点 + 冲突点 = 主标题"） | 做成计算器/生成器 |
| 明确的 before/after 案例 | 切换器 |
| 分阶段的流程描述 | 步骤导航器 |
| "你应该自问"的清单 | 自检打分器 |
| 时间线叙事 | 时间轴 |

---

## 反例（不要做的交互）

- hover 时颜色变深（那是 CSS 装饰，不是交互）
- 点击后只展示一段固定文字（信息密度为零）
- 纯装饰动画（打字机效果、粒子背景）
- tooltip 显示已经能在主文里看到的内容
